Приложение должно содержать следующие данные:
- цель
- задача
- исполнитель

Реализовать API:
- Создание нового пользователя
- Поиск пользователя по логину
- Поиск пользователя по маске имя и фамилии
- Создание новой цели
- Получение списка всех целей
- Создание новой задачи на пути к цели
- Получение всех задач цели
- Изменение статуса задачи в цели

# 2. Определите перечень ролей пользователей и перечень внешних систем.

### Роли
| Роль                                    | Описание                                                                                                             |
|-----------------------------------------|----------------------------------------------------------------------------------------------------------------------|
| Пользователь                            | основной участник сервиса, который может создавать цели, задачи, удалять их и т.д                                    |
| Владелец организации (далее - Владелец) | пользователь, обладающий дополнительными правами для управления пользователями (добавление, удаление из организации) |

Допущение : Организация - единое пространство с целями и задачами, которые видны всем пользователям организации с функционалом
назначения пользователей исполнителями задач.

### Внешние системы

| Система                                 | Назначение                                                                                                             |
|-----------------------------------------|------------------------------------------------------------------------------------------------------------------------|
| Система уведомлений                     | Отправка уведомлений пользователям об изменениях (например, оповещение о том, что пользователя назначили исполнителем) |

# 4. Продумайте основные задачи пользователей и как они могут быть реализованы

#### Задачи пользователей

| Задача                                                     | Описание                                                                                      | API метод                       | Параметры запроса                                                                                       |
|------------------------------------------------------------|-----------------------------------------------------------------------------------------------|---------------------------------|---------------------------------------------------------------------------------------------------------|
| Создание нового пользователя                               | Пользователь может зарегистрироваться в системе, заполнив необходимые поля.                   | `POST /users`                   | - `username` (строка) <br> - `password` (строка) <br> - `firstName` (строка) <br> - `lastName` (строка) |
| Поиск пользователя по логину                               | Пользователь может искать любого другого пользователя по уникальному логину.                  | `GET /users/{username}`         | - `username` (строка)                                                                                   |
| Поиск пользователя по маске имени и фамилии                | Пользователь может выполнять поиск пользователей по частичному совпадению имени и фамилии.    | `GET /users/search`             | - `firstName` (строка) <br> - `lastName` (строка)                                                       |
| Создание новой цели                                        | Пользователь может установить новую цель, к которой будут привязаны задачи.                   | `POST /goals`                   | - `title` (строка) <br> - `description` (строка) <br> - `userId` (строка)                               |
| Получение списка всех целей                                | Пользователь может получить список всех своих целей.                                          | `GET /goals`                    | - `userId` (строка)                                                                                     |
| Создание новой задачи на пути к цели                       | Пользователь может создать задачу, связанную с конкретной целью.                              | `POST /goals/{goalId}/tasks`    | - `goalId` (строка) <br> - `title` (строка) <br> - `description` (строка)                               |
| Получение всех задач цели                                  | Пользователь может получить список всех задач, связанных с конкретной целью.                  | `GET /goals/{goalId}/tasks`     | - `goalId` (строка)                                                                                     |
| Изменение статуса задачи в цели                            | Пользователь может изменять статус выполнения задачи.                                         | `PUT /tasks/{taskId}`           | - `taskId` (строка) <br> - `status` (строка)                                                            |
| Назначение пользователя исполнителем задачи                | Пользователь может назначить другого пользователя исполнителем для конкретной задачи.         | `PUT /tasks/{taskId}/assign`    | - `taskId` (строка) <br> - `userId` (строка)                                                            |
| Удаление исполнителя из задачи                             | Пользователь может удалить исполнителя из конкретной задачи.                                  | `DELETE /tasks/{taskId}/assign` | - `taskId` (строка)                                                                                     |
| Удаление задачи                                            | Пользователь может удалить конкретную задачу из системы.                                      | `DELETE /tasks/{taskId}`        | - `taskId` (строка)                                                                                     |
| Удаление цели                                              | Пользователь может удалить конкретную цель, включая все связанные с ней задачи.               | `DELETE /goals/{goalId}`        | - `goalId` (строка)                                                                                     |

| API Gateway     | Шлюз для обработки API-запросов от клиентского приложения и направления их к соответсвующим сервисам. |
